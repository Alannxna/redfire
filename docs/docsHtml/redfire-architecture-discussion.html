🌟 系统概览

RedFire是一个基于微服务架构的量化交易平台，采用DDD（领域驱动设计）思想，支持多种交易接口，具备完整的风险管理体系。

🎯 核心特性

  • 微服务架构: 服务独立部署，便于扩展
  • 事件驱动: 异步处理，提高性能
  • DDD设计: 清晰的业务边界
  • 高可用性: 服务冗余和故障转移

🏗️ 架构层次

🌐 用户层 (User Layer)

Web用户 + 移动用户 + 专业交易员 + 系统管理员

🎨 前端层 (Frontend Layer)

Web应用 + 移动应用 + 管理后台 + 专业交易界面

🔌 API网关层 (Gateway Layer)

FastAPI网关 + WebSocket服务 + 认证授权 + 负载均衡

⚙️ 后端层 (Backend Layer)

主交易引擎 + 事件引擎 + 策略引擎 + 风险引擎 + 数据引擎

🗄️ 数据层 (Data Layer)

MySQL + Redis + InfluxDB + MongoDB

🌍 外部接口层 (External Layer)

CTP接口 + IB接口 + OKEX接口 + 市场数据源

⚙️ 后端架构

🏗️ 微服务架构

RedFire后端采用集成核心 + 辅助微服务的架构模式：

🔥 VnPy集成核心服务 :8006

功能: VnPy引擎管理、事件处理、网关连接、市场数据、策略管理、Redis消息队列

作用: 系统的核心交易引擎，负责策略执行、订单管理、实时数据处理

👤 用户交易服务 :8001

功能: 用户认证和授权、账户管理、订单交易、风险控制

作用: 处理用户相关的所有业务逻辑，包括认证、交易、风控

📊 策略数据服务 :8002

功能: 策略管理、历史数据管理、回测分析、技术指标计算

作用: 负责策略的创建、管理、回测以及历史数据的处理

🌐 网关适配服务 :8004

功能: CTP接口适配、协议格式转换、订单路由转发

作用: 负责与外部交易接口的连接和协议转换

📱 监控通知服务 :8005

功能: 系统性能监控、服务健康检查、告警规则管理

作用: 负责系统的监控、告警和通知功能
